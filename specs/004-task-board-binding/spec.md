# Feature Specification: Task to Board Binding

**Feature Branch**: `004-task-board-binding`  
**Created**: 2026-03-02  
**Status**: Draft  
**Input**: User description: "Необходимо дать возможность привязывать задачи к доске пользователя. По умолчанию, при создании задачи, мы привязываем ее к доске пользователя."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Автоматическая привязка задачи к доске при создании (Priority: P1)

Пользователь создает новую задачу, и система автоматически привязывает её к его доске по умолчанию. Это обеспечивает, что все задачи пользователя организованы и доступны в контексте его досок.

**Why this priority**: Это базовая функциональность, необходимая для организации задач пользователя. Без автоматической привязки пользователю пришлось бы вручную указывать доску для каждой задачи, что снижает удобство использования.

**Independent Test**: Можно протестировать независимо, создавая новую задачу и проверяя, что она автоматически привязана к доске пользователя по умолчанию. Это доставляет ценность в виде организованного хранения задач.

**Acceptance Scenarios**:

1. **Given** пользователь авторизован и имеет хотя бы одну доску, **When** пользователь создает новую задачу без указания доски, **Then** задача автоматически привязывается к доске пользователя по умолчанию
2. **Given** пользователь авторизован и имеет несколько досок, **When** пользователь создает новую задачу без указания доски, **Then** задача автоматически привязывается к доске, помеченной как "по умолчанию" для этого пользователя
3. **Given** пользователь авторизован и не имеет досок, **When** пользователь создает новую задачу, **Then** система автоматически создает доску по умолчанию и привязывает к ней задачу

---

### User Story 2 - Ручная привязка задачи к конкретной доске (Priority: P2)

Пользователь может явно указать, к какой доске привязать задачу при её создании или редактировании. Это позволяет пользователям организовывать задачи по проектам, категориям или другим критериям.

**Why this priority**: Эта функциональность расширяет возможности организации задач, но не является критической для базового использования, так как автоматическая привязка покрывает основной сценарий.

**Independent Test**: Можно протестировать независимо, создавая задачу с указанием конкретной доски и проверяя, что задача привязана к указанной доске. Это доставляет ценность в виде гибкой организации задач.

**Acceptance Scenarios**:

1. **Given** пользователь авторизован и имеет несколько досок, **When** пользователь создает новую задачу и указывает конкретную доску, **Then** задача привязывается к указанной доске
2. **Given** пользователь авторизован и имеет задачу, привязанную к одной доске, **When** пользователь редактирует задачу и указывает другую доску, **Then** задача перепривязывается к новой доске
3. **Given** пользователь авторизован и пытается привязать задачу к несуществующей доске, **Then** система возвращает ошибку с указанием, что доска не найдена

---

### User Story 3 - Просмотр задач в контексте доски (Priority: P3)

Пользователь может просматривать все задачи, привязанные к конкретной доске. Это позволяет видеть задачи в контексте проекта или категории, к которой относится доска.

**Why this priority**: Эта функциональность улучшает пользовательский опыт, позволяя фильтровать и организовывать задачи, но не является критической для базовой функциональности.

**Independent Test**: Можно протестировать независимо, просматривая список задач на конкретной доске и проверяя, что отображаются только задачи, привязанные к этой доске. Это доставляет ценность в виде удобной навигации по задачам.

**Acceptance Scenarios**:

1. **Given** пользователь авторизован и имеет доску с привязанными задачами, **When** пользователь открывает доску, **Then** отображается список всех задач, привязанных к этой доске
2. **Given** пользователь авторизован и имеет доску без задач, **When** пользователь открывает доску, **Then** отображается пустой список задач
3. **Given** пользователь авторизован и имеет несколько досок с задачами, **When** пользователь переключается между досками, **Then** отображаются только задачи, привязанные к текущей доске

---

### Edge Cases

- Что происходит, когда пользователь пытается привязать задачу к доске другого пользователя?
- Как система обрабатывает ситуацию, когда доска по умолчанию пользователя была удалена?
- Что происходит при попытке создать задачу, когда все доски пользователя были удалены?
- Как система обрабатывает одновременное редактирование привязки задачи несколькими пользователями?
- При удалении доски все привязанные задачи удаляются каскадно

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Система ДОЛЖНА автоматически привязывать новую задачу к доске, которую пользователь явно пометил как "по умолчанию", при создании задачи без указания доски
- **FR-002**: Система ДОЛЖНА позволять пользователю явно указывать доску при создании задачи
- **FR-003**: Система ДОЛЖНА позволять пользователю изменять привязку задачи к другой доске при редактировании
- **FR-004**: Система ДОЛЖНА автоматически создавать доску по умолчанию для пользователя, если у него нет досок при создании первой задачи
- **FR-005**: Система ДОЛЖНА предотвращать привязку задачи к несуществующей доске
- **FR-006**: Система ДОЛЖНА предотвращать привязку задачи к доске другого пользователя (доски принадлежат только одному пользователю)
- **FR-007**: Система ДОЛЖНА хранить информацию о привязке задачи к доске
- **FR-008**: Система ДОЛЖНА предоставлять возможность просматривать все задачи, привязанные к конкретной доске
- **FR-009**: Система ДОЛЖНА обрабатывать ситуацию, когда доска по умолчанию была удалена, используя следующую доступную доску или создавая новую
- **FR-011**: Система ДОЛЖНА удалять все задачи каскадно при удалении доски
- **FR-010**: Система ДОЛЖНА возвращать понятное сообщение об ошибке при попытке привязать задачу к недоступной доске

### Key Entities

- **Задача (Task)**: Представляет единицу работы, которую нужно выполнить. Содержит название, описание, статус и привязку к одной доске (отношение один-ко-многим: одна задача принадлежит только одной доске).
- **Доска (Board)**: Представляет контейнер для организации задач по проектам, категориям или другим критериям. Содержит название, описание и список привязанных задач. Доска принадлежит только одному пользователю.
- **Пользователь (User)**: Представляет пользователя системы, который может создавать доски и задачи. Имеет одну или несколько досок, пользователь может явно помечать одну из них как "по умолчанию".

## Non-Functional Requirements

### Performance & Scalability

- **NFR-001**: Система не имеет жесткого ограничения на количество задач на одной доске; практическое ограничение определяется производительностью инфраструктуры
- **NFR-002**: Время загрузки списка задач на доске не должно превышать 2 секунд для до 1000 задач
- **NFR-003**: Операция создания задачи с автоматической привязкой должна выполняться менее чем за 10 секунд

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Пользователи могут создавать задачи с автоматической привязкой к доске по умолчанию менее чем за 10 секунд
- **SC-002**: 95% задач создаются с корректной привязкой к доске (автоматической или явной)
- **SC-003**: Пользователи могут просматривать задачи в контексте доски с временем загрузки менее 2 секунд
- **SC-004**: 90% пользователей успешно выполняют операцию изменения привязки задачи к другой доске с первой попытки
- **SC-005**: Система корректно обрабатывает все граничные случаи (удаление доски, отсутствие прав доступа) без потери данных

## Clarifications

### Session 2026-03-02

- Q: Какая модель отношений между задачами и досками? → A: Одна задача принадлежит только одной доске (один-ко-многим)
- Q: Что происходит с задачами при удалении доски? → A: Задачи удаляются вместе с доской (каскадное удаление)
- Q: Как определяется доска по умолчанию для пользователя? → A: Пользователь явно помечает одну из своих досок как "по умолчанию"
- Q: Могут ли доски быть общими для нескольких пользователей? → A: Нет, доски принадлежат только одному пользователю
- Q: Какое максимальное количество задач может быть на одной доске? → A: Без жесткого ограничения (практическое ограничение - производительность)
